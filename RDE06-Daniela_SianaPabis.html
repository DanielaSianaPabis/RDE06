SOFTWARES DE VERSIONAMENTO


DEFINIÇÃO

É um sistema que tem por finalidade realizar o controle de versões de um código ou documento. Ademais, o gerenciamento das versões de um código são realizadas diretamente por um desenvolvedor ou uma equipe.

REPOSITÓRIOS: LOCAL E REMOTO

É quase certo que quando se cria um documento/projeto, este necessite de atualizações conforme o tempo.Neste sentido, para que se tenha um controle das novas versões deste documento/projeto criado, é feito por intermédio de um software, um Repositório Local, que criará um histórico de versões, armazenando em um mesmo arquivo as atualizações feitas.
Um Repositório Remoto também é uma opção, quando se está trabalhando de forma conjunta (equipe) ou até mesmo sozinho.

SOFTWARE DE CONTROLE DE VERSÃO

O software de controle de versões gerenciará o histórico de versão local e remoto, de modo a sincroniza-los. Dessa forma, um mesmo arquivo que estará em um repositório, estará em outro.

Exemplo:

Imaginemos dois usuários que estão realizando um mesmo projeto, além disso, ambos estão utilizando o Software de Controle de Versionamento, tendo cada um, portanto, um repositório local. Assim, quando os indivíduos realizam a sincronização dos seus projetos no software, este permitirá ou não as alterações requisitadas pelos usuários, bem como notifica-los. Cria-se então, uma versão corrente que possuirá as alterações feitas pelos usuários e estando disponíveis a estes, evitando assim, erros acidentais, como apagar ou alterar um arquivo.

TIPOS DE SOFTWARE DE CONTROLE DE VERSÃO

Centralizado:

Tem um repositório central em um servidor com controle de versões, ou seja, os usuários entram no servidor e trabalham nele. Além disso, é exclusivamente dependente do servidor, pois se este parar, o controle de versões também irá.

Exemplo de software centralizado:

  • Software subversion

Distribuido:

Possui um repositório local com o controle de versões, ou seja, apenas quando conclui o trabalho é que se fará a atualização no repositório central. Além disso, é independente do servidor.

Exemplos de softwares distribuidos:

  • Software Mercurial
  • Git

GIT

Surgido para o desenvolvimento do Kernel do Linux,o GIT é um Software de Versionamento Livre criado em 2005.
Cada diretório de trabalho deste software é um repositório com um histórico completo, além de possuir um acompanhamento das revisões e não dependender de um servidor central.
